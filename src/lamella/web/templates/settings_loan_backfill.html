Format

One row per payment. Required columns: date, amount. Optional: offset_account, narration.

Date must align with a scheduled payment date the model knows about — the server uses it to look up the amortization split for that month (principal / interest / escrow).

Amount must match the modeled total within $0.02. Larger drift means the row is rejected — usually the result of a partial payment, an extra principal contribution, or a servicer fee that needs special handling.

Offset account

The asset account the payment was drawn from. Per-row override beats the fallback you’ll set on the next step.
