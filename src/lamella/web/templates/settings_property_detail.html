Identity

Property type drives what tax schedule activity rolls into — single-family + primary-residence flows to Schedule A; rental flows to Schedule E.

Entity binding is locked once set. A real-world ownership change (intercompany transfer, sale + repurchase) goes through the disposal flow, not a field edit.

{% endcall %} {% call L.pair_help() %}

Address

Used for property-tax assessor lookups, county-assessor valuation imports, and rental-listing sync. State + postal code are the most load-bearing — they drive depreciation timing rules in a few jurisdictions.

{% endcall %} {% call C.card(title='Purchase') %}
{% endcall %} {% call L.pair_help() %}

Purchase + cost basis

Purchase price + closing costs together form the depreciable basis target. Closing costs include title, transfer tax, and origination fees that capitalize into basis (not points).

Asset account path is the Beancount account that tracks book value. The Cost basis ledger panel above sums postings to this account; it should match the target.

{% endcall %} {% call C.card(title='Flags') %}
{% endcall %} {% call L.pair_help() %}

Flags

Primary residence + rental are independent — a property can be one, both (in the case of an owner-occupied duplex), or neither (vacant lot, second home). Tax flows fork on these flags.

Active only flips false when the property is fully closed out — sold, foreclosed, or otherwise disposed.
