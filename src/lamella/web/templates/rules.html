Probably not. The AI handles classification using every context signal — entity descriptions, account descriptions, active notes, receipt content, mileage logs, active projects, vector-similar history, merchant frequency histograms. Your job is usually to write context (on Entities, Accounts, Projects), not rules.

Rules exist for one case: a merchant so unambiguous, recurring, and well-scoped that running the AI on every occurrence is a waste of tokens. Example: a Netflix subscription on a specific card always goes to one account. A hard rule at confidence ≥ 0.95, card-scoped, saves the classify call.

Don’t make a rule for a merchant whose correct target depends on context. Hardware Store could be inventory, supplies, or a project. The AI resolves those case-by-case using receipt content + active projects + mileage; a hard rule would silently miscategorize.

AI-learned rules never auto-apply; only user-created rules at confidence ≥ 0.95 skip the classify call. The intercompany gate still overrides auto-apply when a wrong-card situation is detected.
